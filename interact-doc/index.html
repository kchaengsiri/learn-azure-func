Architecture Insight

In this model, the **Azure Function is tightly coupled** to the number of downstream systems. If you add a "Loyalty Program" worker later, you have to update the Function code to write to a 5th queue.

Architecture Insight

This is the **Publisher/Subscriber (Pub/Sub)** pattern. The Function doesn't care who is listening. You can add new consumers (like a Data Warehouse or Mobile Alert) just by adding a new Subscription in Azure—no code changes required!

Why Two Stages?

1. Immediate Satisfaction: As soon as the user hits "Donate", you send a Thank You email via Service Bus Topic. This is fast and reliable.

2. Complex Coordination: Generating a Receipt and a Certificate takes time. You don't want to send 3 separate emails. Instead, a Durable Orchestrator waits for both PDF tasks to finish, then sends one final email with everything attached.

At-a-glance Comparison

Feature Azure Queue Storage Azure Service Bus
Model 1-to-1 (Point-to-Point) 1-to-Many (Pub/Sub via Topics)
Ordering (FIFO) Best effort (Not guaranteed) Strictly guaranteed (via Sessions)
Delivery Guarantee At-Least-Once At-Least-Once or At-Most-Once
Receive Mode Polling (Non-blocking) Long Polling & Push-style
Max Queue Size Up to 5 PiB (Storage Account limit) 1 GB to 80 GB (per entity)
Message Size 64 KB max 256 KB to 100 MB (Tier dependent)
Transactions No Yes (Local & Atomic)
Duplicate Detection No Yes (based on MessageId)
Dead-Lettering No (Manual handling) Yes (Built-in $DeadLetterQueue)

Use Queue Storage if...

  • You need to store massive amounts of messages (> 80 GB / up to 5 PiB).
  • Your application requires an "At-Least-Once" model without complex features.
  • Lowest cost for basic buffering is the primary concern.
  • You need to track progress via DequeueCount.

Use Service Bus if...

  • Your Donation Use Case: Multiple tasks must trigger from one event.
  • Strict FIFO ordering or Sessions are required.
  • You need "Exactly-Once" processing patterns.
  • Built-in features like Duplicate Detection or Dead-Lettering are needed.

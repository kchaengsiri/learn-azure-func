Virtual Actor Model

Entities are created automatically the first time they are called. They "sleep" when not in use and "wake up" to process operations, preserving state forever.

Key Operations

  • Signal: One-way fire-and-forget call.
  • Call: Two-way request/response (awaits result).
  • Locking: Entities can be locked for atomic multi-entity operations.
